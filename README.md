## How I work

- **Value case first.** Every project starts from a business problem and a metric, not a model.
- **Trust is a feature.** Source grounding, eval harnesses, audit trails, and explicit scope
  boundaries ("this tool does not underwrite") appear in the product, not just the docs.
- **Ship to learn.** Zero-backend, no-build prototypes that stakeholders can open and react
  to in days, not quarters.
